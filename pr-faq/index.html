Защита второй лабы по проге

Сами задания были несложными:

Задание 1.

В директории есть файлы program.c; program.h; f.h; f.c. Известно, что в program.h подключен файл f.h. Выберите правильную команду для того, чтобы получить исполняемый файл. Функция main определена в файле program.c. Выберите один ответ:

  • gcc -c program.c f.c
  • gcc -c program.c
  • gcc program.c f.c
  • gcc program.c f.h

Задание 2.

#include
#define PROD<пропуск>

int main() {
    int a = 10;
    printf("%d",PROD(10,4));
    return 0;
}
Что необходимо прописать на месте <пропуск>, чтобы программа выводила 40. Выберите один ответ:
  • a * b
  • (a,b) (a * b)
  • a,b (a*b)
  • Ни один вариант не правильный

Задание 3.

Напишите программу, которая принимает на вход массив целых чисел длины не более 20. В массиве замените все числа на 0, если сумма всех чисел в исходном массиве после него была четной. Например, массив {1 3 1 2 1 3} должен преобразоваться к следующему {0 3 0 0 1 3}. Для последнего элемента сумма не считается

Тест Ввод Результат 
1
 
9 -33 4 -42 -12 -36 15 -39 -3 29 -6 -14 0 34 18 -3 -35 -49 -11 -27
 
0 -33 4 -42 -12 -36 0 -39 0 29 -6 -14 0 34 18 0 -35 0 -11 -27

А сам преподавать спрашивал мощно: этапы сборки программы, что делает пропроцессинг, что такое объектный файл, зачем объектные файлы вообще придумали, что такое линковка, зачем она придумана и что она вообще делает, какие директивы препроцессора существуют, как можно избежать повторного подклюения файла и следовательно повторного объявления чего-либо. Про саму лабу он почти не спрашивал.
